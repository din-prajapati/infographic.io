# Prompt Engineering Strategy — InfographicAI
**Version:** 1.1 | **Date:** 2026-04-29  
**Scope:** AI image generation pipeline — OpenAI Vision + Ideogram/Nano Banana prompt design  
**Goal:** Define current state, rank it, and specify the innovative target architecture by phase  
**Update v1.1:** Added multi-modal input layer — property photos, agent headshot, brokerage logo

## Part 1: Current Approach Audit

### 1.1 What the Pipeline Does Today

The image generation pipeline is a **2-call + 1 template** architecture:

```
User types prompt
  → PromptExtractor (OpenAI) — extracts fields from natural language
  → analyzeProperty() (OpenAI gpt-5) — generates a headline string
  → generateImagePrompt() (NO LLM) — pure string interpolation template
  → applyStylePreset() — appends style modifier to end of string
  → getVariationModifier() — appends "slightly different color scheme"
  → Ideogram API — generates the image
```

**Critical insight:** The image prompt sent to Ideogram is **not generated by an LLM**. It is a static developer-written template with variable slots. The model never reasons about what the property needs visually.

### 1.2 Actual Prompts in Production

**Step 1 — OpenAI headline (openai.service.ts:29-38):**
```
Analyze this real estate property and create a compelling headline (max 50 chars):

Property: ${address}
Type: ${propertyType}
Price: $${price}
Beds: ${beds}, Baths: ${baths}
Sqft: ${sqft}
Features: ${features}

Create a headline that highlights the most appealing aspect. Be concise and engaging.
```

**Step 2 — Image prompt (openai.service.ts:55-64) — this is a template, not an LLM call:**
```
Professional real estate infographic design:
- Main text: "${headline}"
- Property: ${address}
- Price: $${price} prominently displayed
- ${beds} bed, ${baths} bath, ${sqft} sqft
- Agent: ${agentName}${brokerage}
- Brand colors: ${colors}          ← hex values Ideogram cannot interpret
- Clean, modern, luxury aesthetic
- High contrast, readable typography
- Professional layout with clear hierarchy
```

**Step 3 — Style append (ai-orchestrator.service.ts:220-231):**
```
${prompt}. Style: Modern, minimalist design with clean lines and contemporary typography
```

**Step 4 — Variation modifier (ai-orchestrator.service.ts:234-242):**
```
Variation 2: alternative layout arrangement   ← too vague, Ideogram ignores it
```

### 1.3 Data Injection Audit

#### Text Fields

| Data Field | Reaches Image Prompt? | How |
|-----------|----------------------|-----|
| Address | Yes | Template slot |
| Price | Yes | Formatted currency string |
| Beds / Baths / Sqft | Yes | Number interpolation |
| Agent Name | Yes | Template slot |
| Brokerage | Yes | Appended after name |
| Brand Colors | Partial | Hex strings — Ideogram reads as text, not color |
| Property Features | **No** | Extracted but silently dropped |
| Listing Type | **No** | for_sale / for_rent ignored |
| Property Type | **No** | Residential / commercial ignored |
| Agent Phone / Email | **No** | UI collects it, never reaches prompt |
| Price Tier / Segment | **No** | $100K starter treated identically to $5M penthouse |
| Buyer Persona | **No** | Not computed |
| Neighborhood Context | **No** | Address only as text |

#### Image / Visual Assets — The Largest Gap

| Asset | DTO Field | UI Collects? | Used in Generation? | Impact of Gap |
|-------|-----------|-------------|---------------------|---------------|
| Property photos (exterior, interior, aerial) | `photos?: string[]` | Yes — `ImageUploadPanel` exists with `onImageUpload` callback | **No** — `handleImageUpload` is a stub (`console.log` only); field never passed to backend | AI generates a generic property scene instead of the actual property |
| Agent headshot / profile photo | Not in DTO (only `logoUrl`) | Partially — `AgentInfoForm` has photo upload area | **No** — field not in DTO, never reaches backend | Infographic shows no real agent face |
| Brokerage logo | `logoUrl?: string` in `AgentBrandingDto` | Yes — AgentInfoForm collects it | **No** — received by DTO but never read in `openai.service.ts` or `ai-orchestrator.service.ts` | Logo never appears in infographic |
| Style reference image | None | `ImageUploadPanel` hint text mentions it | **No** | User cannot guide visual style with a reference |

**The net result:** A real estate agent uploads 5 property photos and their headshot — none of it affects the generated image. The infographic shows a generic AI-rendered house, not their actual listing.

## Part 2: Innovative Target Architecture

### 2.1 Core Design Principle

> **One template cannot serve all use cases. The prompt should be reasoned, not templated.**  
> A luxury penthouse requires aspirational editorial language. A studio rental needs lifestyle-forward warmth. A commercial office needs data-forward professionalism. These cannot share a prompt.

### 2.2 Generation Strategy — Asset-Driven Branching

Before any prompt is written, the pipeline classifies which generation strategy to use based on what visual assets the user has provided. These are fundamentally different generation modes, not variations of the same prompt.

```
Input asset inventory:
  Has property photos?  Has agent headshot?  Has logo?
         │                     │                │
         ▼                     ▼                ▼
  ┌──────────────────────────────────────────────────────┐
  │ Strategy A: Text-Only (no images)                    │
  │   → Full AI text-to-image via Nano Banana            │
  │   → Generic property scene — no reference            │
  │   → Current approach (improved with better prompts)  │
  ├──────────────────────────────────────────────────────┤
  │ Strategy B: Property-Photo-Guided                    │
  │   → Stage 0: GPT-4o Vision analyzes photos          │
  │   → Extracts: style, colors, standout features      │
  │   → Stage 3: Pass photos as reference images         │
  │     to Nano Banana Pro (up to 14 references)         │
  │   → Generated scene reflects actual property         │
  ├──────────────────────────────────────────────────────┤
  │ Strategy C: Full-Composite (all assets present)      │
  │   → Strategy B for base infographic generation       │
  │   → Stage 4.5: Post-process compositing:             │
  │     • Agent headshot → circular crop → lower-left   │
  │     • Logo → transparent overlay → lower-right      │
  │     • Brand color accent bars composited in          │
  │   → Final image is real estate publication quality   │
  ├──────────────────────────────────────────────────────┤
  │ Strategy D: Photo-as-Canvas (Phase 3)                │
  │   → Property photo IS the infographic background    │
  │   → Nano Banana edits the photo: "add infographic   │
  │     overlay with these details to this image"        │
  │   → Highest authenticity — actual listing shown     │
  └──────────────────────────────────────────────────────┘
```

### 2.3 Six-Stage Pipeline (Full Target State)

```
Stage 0: Multi-Modal Input Classification  What assets do we have? Which strategy?
Stage 0.5: Vision Analysis                 What do the property photos actually show?
Stage 1: Property Intelligence             Who is this listing for? What matters most?
Stage 2: Brand Context Assembly            What is the agent's complete visual identity?
Stage 3: LLM-Generated Image Prompt       AI designs the visual, enriched by all stages above
Stage 4: Variation Differentiation        3 structurally different compositions
Stage 4.5: Post-Process Compositing       Agent headshot + logo overlaid on generated base
```

### 2.4 Stage 0 — Multi-Modal Input Classification

**Goal:** Inspect what the user uploaded and route to the right generation strategy.

```typescript
interface AssetInventory {
  propertyPhotos: string[];      // URLs from ImageUploadPanel / photos[] DTO field
  agentPhotoUrl: string | null;  // Agent headshot (new DTO field: agent.photoUrl)
  logoUrl: string | null;        // Brokerage logo (existing DTO field: agent.logoUrl)
  styleReference: string | null; // Optional aesthetic reference image
}

function classifyGenerationStrategy(assets: AssetInventory): GenerationStrategy {
  const hasPropertyPhotos = assets.propertyPhotos.length > 0;
  const hasAgentAssets = !!(assets.agentPhotoUrl || assets.logoUrl);

  if (!hasPropertyPhotos && !hasAgentAssets) return 'TEXT_ONLY';       // Strategy A
  if (hasPropertyPhotos && !hasAgentAssets)  return 'PHOTO_GUIDED';    // Strategy B
  if (hasPropertyPhotos && hasAgentAssets)   return 'FULL_COMPOSITE';  // Strategy C
  if (!hasPropertyPhotos && hasAgentAssets)  return 'BRAND_COMPOSITE'; // Logo/headshot on AI scene
}
```

### 2.7 Stage 2 — Brand Context Assembly

**Goal:** Transform raw agent form data AND agent visual assets into a complete visual brand specification. This is the layer that makes every infographic feel branded to a specific agent.

```typescript
interface BrandContext {
  // Color intelligence
  primaryColor: string;           // hex → descriptive language
  accentColor: string;
  colorMood: string;

  // Agent visual presence
  agentPresence: 'PROMINENT' | 'BRANDED' | 'MINIMAL';
  agentPhotoUrl: string | null;   // NEW: headshot URL for compositing
  logoUrl: string | null;         // Logo URL for compositing
  hasCompositingAssets: boolean;  // Signals Stage 4.5 should run

  // Trust signals
  trustSignals: string[];
  primaryContact: string | null;

  // Compositing layout decision
  compositingLayout: CompositingLayout | null;
}

function buildBrandContext(agent: AgentProfile): BrandContext {
  const agentPhotoUrl = agent.photoUrl ?? null;    // NEW DTO field
  const logoUrl = agent.logoUrl ?? null;           // Existing DTO field (now actually used)

  return {
    primaryColor: hexToColorDescription(agent.brandColors[0]),
    accentColor: hexToColorDescription(agent.brandColors[1] ?? '#FFFFFF'),

    // Agent prominence: photo > name > none
    agentPresence:
      agentPhotoUrl ? 'PROMINENT' :
      agent.name    ? 'BRANDED'   :
                      'MINIMAL',

    agentPhotoUrl,
    logoUrl,
    hasCompositingAssets: !!(agentPhotoUrl || logoUrl),

    trustSignals: [
      agent.license   && `Licensed Agent: ${agent.license}`,
      agent.brokerage && agent.brokerage,
      agent.website   && 'Verified Online Presence',
    ].filter(Boolean) as string[],

    primaryContact: agent.phone || agent.email || agent.website || null,

    // Determine compositing layout based on available assets
    compositingLayout: buildCompositingLayout(agentPhotoUrl, logoUrl),
  };
}
```

**Compositing layout decision tree:**
```typescript
type CompositingLayout =
  | 'AGENT_LEFT_LOGO_RIGHT'    // Both assets: agent photo left, logo right in lower third
  | 'AGENT_CENTER'             // Agent photo only: centered in lower third
  | 'LOGO_ONLY'                // Logo only: lower-right corner watermark
  | 'TEXT_ONLY';               // No visual assets: name/contact text only

function buildCompositingLayout(
  agentPhotoUrl: string | null,
  logoUrl: string | null
): CompositingLayout {
  if (agentPhotoUrl && logoUrl) return 'AGENT_LEFT_LOGO_RIGHT';
  if (agentPhotoUrl)            return 'AGENT_CENTER';
  if (logoUrl)                  return 'LOGO_ONLY';
  return 'TEXT_ONLY';
}
```

**Hex-to-description mapping (covers common real estate brand colors):**
```typescript
function hexToColorDescription(hex: string): string {
  const colorMap: Record<string, string> = {
    '#1F448B': 'deep navy blue, professional and trustworthy',
    '#FFFFFF': 'pure white, clean and modern',
    '#C8A96E': 'warm champagne gold, luxury and refinement',
    '#2E7D32': 'forest green, growth and nature',
    '#B71C1C': 'bold red, urgency and energy',
    '#212121': 'rich charcoal black, sophisticated and premium',
    '#F5F5F5': 'light warm grey, approachable and clean',
    '#8B4513': 'warm saddle brown, earthy and grounded',
    '#4A4A4A': 'sophisticated dark grey, modern and premium',
  };
  return colorMap[hex.toUpperCase()] ?? describeHexByHSL(hex);
}

// Fallback: analyze hue/saturation/lightness and describe in plain language
function describeHexByHSL(hex: string): string {
  const { h, s, l } = hexToHSL(hex);
  const hueLabel =
    h < 30  ? 'warm red-orange' : h < 60  ? 'golden amber'  :
    h < 90  ? 'yellow-green'    : h < 150 ? 'fresh green'   :
    h < 180 ? 'teal'            : h < 240 ? 'cool blue'     :
    h < 270 ? 'indigo'          : h < 300 ? 'purple'        : 'warm rose';
  const lightLabel = l > 80 ? 'light' : l < 30 ? 'deep' : 'rich';
  const satLabel   = s < 20 ? 'muted, neutral' : s > 70 ? 'vibrant' : 'balanced';
  return `${lightLabel} ${hueLabel}, ${satLabel}`;
}
```

### 2.10 Stage 4.5 — Post-Process Compositing (Phase 2)

**Goal:** Overlay real agent assets onto the generated infographic. This is image processing, not AI — it's fast, cheap, and produces professional results.

**Why compositing instead of including in the AI prompt:**
- AI image generators cannot faithfully reproduce a specific real person's face from a URL. They generate a plausible face, not the actual agent.
- Logo marks require pixel-perfect reproduction. AI generators approximate logos at best.
- Compositing (sharp/jimp) is deterministic, fast (~50ms), and produces exact results.

**Compositing pipeline (runs after Stage 4 image generation completes):**

```typescript
// In: compositing.service.ts (Phase 2 — uses sharp library)
async function compositeAgentAssets(
  generatedImageUrl: string,
  brand: BrandContext,
): Promise<string> {

  const baseImage = await sharp(await fetchBuffer(generatedImageUrl));
  const { width, height } = await baseImage.metadata();

  const composites: sharp.OverlayOptions[] = [];

  // 1. Agent headshot — circular crop → lower-left
  if (brand.agentPhotoUrl) {
    const headshot = await prepareCircularHeadshot(brand.agentPhotoUrl, {
      diameter: Math.round(height * 0.12),     // 12% of infographic height
      borderColor: brand.primaryColorHex,
      borderWidth: 3,
    });
    composites.push({
      input: headshot,
      top: Math.round(height * 0.865),         // Bottom 13.5% zone
      left: Math.round(width * 0.04),
    });
  }

  // 2. Brokerage logo — lower-right
  if (brand.logoUrl) {
    const logo = await prepareLogo(brand.logoUrl, {
      maxWidth: Math.round(width * 0.20),      // Max 20% of infographic width
      maxHeight: Math.round(height * 0.08),
      background: 'transparent',
    });
    composites.push({
      input: logo,
      top: Math.round(height * 0.88),
      gravity: sharp.gravity.southeast,        // Pin to lower-right
    });
  }

  // 3. Brand color accent bar behind agent section
  if (brand.agentPhotoUrl || brand.logoUrl) {
    const accentBar = await sharp({
      create: {
        width,
        height: Math.round(height * 0.15),
        channels: 4,
        background: { ...hexToRGB(brand.primaryColorHex), alpha: 200 },
      }
    }).png().toBuffer();

    composites.unshift({              // Insert FIRST so it's behind agent/logo
      input: accentBar,
      top: Math.round(height * 0.85),
      left: 0,
      blend: 'over',
    });
  }

  // Composite all layers and upload to R2
  const finalBuffer = await baseImage.composite(composites).jpeg({ quality: 92 }).toBuffer();
  return await uploadToR2(finalBuffer, `infographics/${infographicId}/final.jpg`);
}
```

**Compositing layouts by asset availability:**

```
AGENT_LEFT_LOGO_RIGHT:          AGENT_CENTER:              LOGO_ONLY:
┌─────────────────────┐         ┌─────────────────────┐   ┌─────────────────────┐
│                     │         │                     │   │                     │
│   Property Image    │         │   Property Image    │   │   Property Image    │
│                     │         │                     │   │                     │
│  ─ ─ ─ ─ ─ ─ ─ ─  │         │  ─ ─ ─ ─ ─ ─ ─ ─  │   │  ─ ─ ─ ─ ─ ─ ─ ─  │
│ [●] Name · Contact  [Logo] │  │    [●]  Name · Contact  │   │ Name · Contact    [Logo] │
└─────────────────────┘         └─────────────────────┘   └─────────────────────┘
  Agent left, logo right          Agent centered              Logo lower-right
```

### Current Models
- **Headline:** OpenAI `gpt-5` (correct per knowledge cutoff Aug 2025)

### Phase 0.5 — No model changes (fix prompts first, change models in Phase 1)

### Phase 1 — Adopt Nano Banana (from ARCHITECTURE_GAP_ANALYSIS.md Phase 0.5 plan)

From the scratch research, Nano Banana Pro (Gemini 3 Pro Image) outperforms Ideogram for infographics specifically because of:
- **Superior text rendering** — price, address, agent name are legible
- **Search-grounded generation** — can pull neighborhood context
- **Multi-turn editing** — "make the header bolder" works natively

| Plan Tier | Image Model | Why | Cost/image |
|-----------|------------|-----|------------|
| FREE | Nano Banana Flash | Lowest cost | $0.002-0.008 |
| SOLO | Nano Banana Flash 2K | Better text than Ideogram | $0.008 |
| TEAM | Nano Banana Pro 2K | Premium quality | $0.050 |
| BROKERAGE | Nano Banana Pro 4K | Maximum quality | $0.100 |

**Cost delta:** FREE tier drops from $0.025 (Ideogram Turbo) to $0.002-0.008 (Nano Flash). TEAM tier goes from $0.080 (Ideogram V2) to $0.050 (Nano Pro 2K). Costs improve at every tier.
