## Conflict Policy

1. **Rebase, don't merge** — keep history linear before squashing.
   ```bash
   git fetch origin
   git rebase origin/{default_branch}
   ```
2. **Resolve conflicts in editor**, never with `--strategy ours/theirs` on shared files.
3. **Force-push to your story branch is allowed** (it's a single-author story branch).
   Never force-push to `{default_branch}`.

## What NOT to do

- ❌ Skip hooks (`--no-verify`, `--no-gpg-sign`) — fix the underlying issue
- ❌ Commit `.env` or secrets (see `sensitive_paths` in PROJECT_CONTEXT.yaml)
- ❌ Mix stories in one PR
- ❌ Commit without the `US-{DOMAIN}-{NNN}` tag (breaks the cross-link)
- ❌ Edit a published commit (use a new commit; `--amend` is for in-flight pre-push work only)

## Hooks (project)

ORION ships shell-only hooks under `.claude/hooks/`. Notable:

- `pre-commit-typecheck` — runs `npm run check` before commit
- `pre-edit-env-secret` — blocks edits to `sensitive_paths`
- `stop-session-summary` — appends session log to TEAM_STATUS.md

Don't disable these globally. If a specific story needs an exception, document it in STORY.md
under "DoD exception" with a written reason and approval.
